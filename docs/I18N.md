# 工具箱国际化（core/i18n）

> 状态：**已落地**（common 原子 + page 页面层 + `Join-I18nParts`）

## 一、文件与 API

| 路径 | 说明 |
|------|------|
| `package/core/i18n/zh.json`、`en.json` | 工具箱文案；语种 = 目录下 `*.json` 文件名 |
| `package/core/lib/config/I18n.ps1` | 加载、`Get-I18n`、`Join-I18nParts`、`Get-ToolkitI18n` |

| API | 用途 |
|-----|------|
| `Get-I18n -Key '...' -Vars @{ ... }` | 工具箱文案（支持 `{shortName}` 等 manifest 模板变量） |
| `Get-I18nRaw -Key '...'` | 原始模板字符串 |
| `Join-I18nParts -Keys @('common.action.navSelect', ...)` | 用 `common.hintSeparator` 拼接底栏 hint |
| `Get-MenuListHintLine` | 列表页默认 hint 行 |
| `Get-InstallDepsFooterHint` | 依赖专页底栏 hint |
| `Get-ToolkitI18n` / `Get-ToolkitI18nRaw` | **供工具脚本**引用工具箱公共文案（与 `Get-I18n` 等价） |
| `Get-ToolI18n` / `Get-ToolI18nRaw` | 读 `tools/<id>/i18n/{locale}.json` |

**开发/预览**（`MIAO_DEV`、`[预览]` 等）不进 i18n，在代码里写死中文即可。

## 二、JSON 分区

```
brand / header / panel     品牌与顶栏
common.action              原子操作（↑↓ 选择、Enter 确认、Esc 退出…）
common.nav                 快捷键（S 设置、H 帮助）
common.label               通用标签（版本、发布日期）
common.status              通用状态（已安装、完成、失败…）
common.unit                计数单位（项、个）
common.pagination          分页（第 n/m 页、共 x 项）
common.cmd                 命令说明（help 页与各工具共用）
common.hintSeparator       hint 片段分隔符（zh: " |  "，en: " · "）
flash.*                    菜单闪提示
error.* / warn.*           错误与警告
shell.*                    仅 Shell 特有（如 exitConfirmPrompt）
page.home / page.settings / page.update / page.install / page.help / page.lang / page.afterTool / page.toolDeps
```

**原则：** 整句底栏 hint 用 `Join-I18nParts` 组合 `common.action.*`，不在 JSON 里再写一条长 `hintSelect`。

## 三、工具侧约定

- 工具展示名、工具内菜单 → `tools/<id>/i18n/{locale}.json`（见 [TOOL-CONVENTION.md](TOOL-CONVENTION.md)）。
- 与工具箱相同的操作文案（如 `Enter 确认`）在工具脚本中调用：

```powershell
Get-ToolkitI18n -Key 'common.action.confirmEnter'
```

- 不要复制 `common.*` 到工具 json；工具 json 只放本工具专属文案。

## 四、新增语种

1. 在 `core/i18n/` 增加 `{locale}.json`（结构与 `zh.json` 一致）。
2. 在 `page.lang.locale` 下增加显示名，例如 `"ja": "日本語"`。
3. 用户通过 `miao lang` 或 `config.json` 的 `locale` 切换。
