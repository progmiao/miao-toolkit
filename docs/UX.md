# 交互与用户体验（UX）

> 状态：**已冻结**（第 8 步确认：2026-05-24）

## 一、主菜单

`miao` 无参数时进入工具列表：

```
 程序喵 Miao 工具包
 ↑↓ 选择   Enter 进入   Esc 退出
------------------------------------------------
 > node     Node.js 版本管理
------------------------------------------------
 共 1 个工具
```

**UI 规范（与 node 工具菜单一致）：**

- 固定视口头尾，列表区域固定高度、原地刷新
- 禁止每次按键 `Clear-Host` 整屏重绘
- 视口内移动只更新两行（旧选中 / 新选中）
- 菜单期间隐藏光标

**同一 UI 规范适用于：**

- `miao` 主工具列表
- `miao install` 依赖多选列表（含 `[未安装]` / `[已安装 · 最新]` / `[可更新]` 三态，见 [INSTALL.md](INSTALL.md)）
- 各工具内部菜单（如 `miao node` 版本列表）

### 版本可用性提示与菜单衔接

本会话 **第一次** `miao` 若检测到工具箱有新版本，在 **菜单或输出上方** 打印一行提示（不阻塞、不确认），随后正常显示主菜单。详见 [UPDATE.md §六](UPDATE.md#六版本可用性提示后台检查)、[CLI.md §一](CLI.md#入口行为miaops1)。

```
[!] Miao 有新版本 v0.2.0（当前 v0.1.0）。运行 miao update 或 winget upgrade miao 更新。

 程序喵 Miao 工具包
 ↑↓ 选择   Enter 进入   Esc 退出
 ...
```

## 二、两种进入方式

| 进入方式 | 典型场景 | 工具结束后 |
|----------|----------|------------|
| `miao` → 选择工具 | 探索、连续使用多个工具 | **询问** 返回列表或退出 |
| `miao node` 直达 | 明确任务、脚本调用 | **直接退出**，不询问 |

## 三、工具结束后的返回行为（UX 推荐）

### 问题

从 `miao` 菜单进入工具后，默认自动返回列表是否友好？

### 推荐方案：**不自动返回，每次让用户选择**

工具执行完毕后，在输出下方显示：

```
────────────────────────────────
[Enter] 返回工具列表
[Q]     退出 Miao
```

| 按键 | 行为 |
|------|------|
| Enter | 回到 `miao` 主工具列表 |
| Q / q | 退出，回到 PowerShell 提示符 |
| Esc | 同 Q，退出 |

### 为何这样设计

| 方案 | 优点 | 缺点 |
|------|------|------|
| 自动返回列表 | 操作少 | 用户可能还没看完安装输出就被切走 |
| 自动退出 | 适合脚本 | 从菜单进入时体验割裂 |
| **询问后选择（推荐）** | 用户可控；可读完输出再决定 | 多按一次键 |

**结论：**

- 从 **菜单进入** → 结束后 **询问**（Enter 回列表 / Q 退出）
- 从 **直达命令** → 结束后 **直接退出**，不询问
- **默认不高亮**「返回」或「退出」，两个选项并列，避免暗示必须返回

### 后续可选增强（非 v1 必须）

用户可在 `%APPDATA%\Miao\config.json` 设置偏好：

```json
{
  "afterTool": "ask"
}
```

可选值：`ask`（默认）| `list`（总是回列表）| `exit`（总是退出）

v1 仅实现 `ask`；配置项写入 DESIGN 预留，不阻塞首版。

## 四、主菜单 Esc 行为

在主工具列表按 Esc → 直接退出 Miao，无需二次确认。

## 五、错误与中断

| 场景 | 行为 |
|------|------|
| 工具内 Esc | 视为取消当前工具，回到「返回 / 退出」提示（菜单进入时） |
| 依赖安装失败 | 显示原因，询问是否仍进入工具 / 返回列表 |
| Ctrl+C | 退出 Miao，返回 PowerShell |

## 六、输出可读性

- 工具内重要操作用颜色区分（信息 / 成功 / 警告 / 错误）
- 返回提示前留空行，与工具输出分隔
- 从菜单再次进入工具时 `Clear-Host` 一次，避免与上一轮输出混杂

## 七、与 help 的配合

用户在菜单或工具失败后，提示：

```
可使用 miao help node 查看详细说明
```

降低「不知道怎么办」的挫败感。
